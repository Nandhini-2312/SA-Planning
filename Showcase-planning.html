Day 3 · Thursday
Customer Form & List View
D3
Thursday — Customers Module
Input Forms · INSERT Query · List View · Edit Functionality
Web OnlyNo UI Polish
Morning Reading — Before Opening the Builder
Required
fieldproxy-input-forms.md — read fully. Understand what each input_type stores and how values are accessed.
fieldproxy-actions.md — focus on execute_query, update_data, and open_popup specifically.
First Half — Basic Form
Create a Customer Form with 3 fields: First Name, Last Name, Mobile Number
All input_type: "text" — no dropdowns yet. Plain form, no styling.
Save button with execute_query → INSERT into customers table
Use NOW() for created_at. Verify the row appears in the database directly after saving.
Create a Customer List View — query all customers, display in a basic list widget
Show full name and mobile number. No sorting or search yet.
After save, use update_data to refresh the list — not refresh_data
Second Half — More Fields + Edit
Add Email and Address fields to the form
Email uses input_type: "email". Address uses input_type: "text" with max_lines: 2.
Add Edit Customer — clicking a list row opens a popup pre-filled with that customer's data
Pre-fill using arguments passed from the row click. Save triggers an UPDATE query on the correct row.
Update the list view to display all 5 fields
End of Day Deliverable
Customer form saves correctly to the DB (verify in Supabase directly)
List view shows all saved customers with live refresh after each save
Edit popup pre-fills data correctly and updates the right row on save

Day 5 · Tuesday
Technician Assignment + n8n Automation
D5
Tuesday — Assignment + Automation
Technician Dropdown · n8n Webhook · Email Notification on Job Creation
n8n IntegrationGate Day
Morning Reading
Required
fieldproxy-actions.md — study the api_call action type. This is what triggers the n8n webhook from FieldProxy.
FieldProxy Tasks
Add Technician dropdown to the Job form — query technicians table, display name, store id
Add Assign Technician button on Job Detail — opens a popup to reassign the technician
UPDATE jobs SET technician_id = ... WHERE id = {{arguments.job_id}}
Update Jobs list to show the assigned technician name alongside the job status
n8n Workflow — Job Created → Customer Email
Webhook Trigger
→
Set Node (format data)
→
Send Email
Build Steps
01
In n8n: create a new workflow. Add a Webhook node — copy the Test URL to use during development.
02
Add a Set node to extract: job_title, customer name, and status from the incoming payload.
03
Add a Send Email node (Gmail or SMTP). Subject: "New Job Created: {{job_title}}". Body: customer name + job details.
04
In FieldProxy — on the Create Job button's onclick, add a second action after the INSERT: type: "api_call" with method POST, url = n8n webhook URL, body = job data as JSON.
05
Test end-to-end: create a job in FieldProxy → check n8n execution log → confirm email is received.
Important: n8n work only begins once the FieldProxy app (customers + jobs) is fully working from Days 3–4. Do not debug two systems simultaneously.
End of Day Deliverable
Technician can be assigned to a job from the detail view
Creating a job fires the n8n webhook and an email is received
n8n execution log confirms a successful workflow run
